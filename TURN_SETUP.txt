TURN setup (coturn) + app env config

1) Install coturn (Ubuntu 20.04+)
   sudo apt update
   sudo apt install -y coturn

2) Open firewall
   UDP/TCP 3478 (TURN/STUN)
   UDP 49152-65535 (relay range)

3) Configure coturn
   Edit /etc/turnserver.conf with:

   listening-port=3478
   tls-listening-port=5349
   fingerprint
   use-auth-secret
   static-auth-secret=CHANGE_ME_SECRET
   realm=YOUR_DOMAIN_OR_IP
   total-quota=100
   bps-capacity=0
   stale-nonce
   no-loopback-peers
   no-multicast-peers
   relay-ip=YOUR_PUBLIC_IP
   external-ip=YOUR_PUBLIC_IP
   min-port=49152
   max-port=65535
   log-file=/var/log/turn.log
   simple-log

4) Enable and start coturn
   sudo sed -i 's/^#TURNSERVER_ENABLED=.*/TURNSERVER_ENABLED=1/' /etc/default/coturn
   sudo systemctl enable coturn
   sudo systemctl restart coturn
   sudo systemctl status coturn

5) Generate TURN credentials (time-limited)
   Format: username = timestamp:userid
   HMAC-SHA1(secret, username) -> password (base64)
   Example helper (Python):
     python3 - <<'PY'
     import hmac, hashlib, base64, time
     secret = b'CHANGE_ME_SECRET'
     username = f"{int(time.time())+3600}:user".encode()
     password = base64.b64encode(hmac.new(secret, username, hashlib.sha1).digest()).decode()
     print('username=', username.decode())
     print('password=', password)
     PY

6) App env variables (server process)
   QBE_TURN_URLS=turn:YOUR_DOMAIN_OR_IP:3478?transport=udp,turn:YOUR_DOMAIN_OR_IP:3478?transport=tcp
   QBE_TURN_USERNAME=<generated username>
   QBE_TURN_CREDENTIAL=<generated password>

Notes
- Use HTTPS on your web server, otherwise getUserMedia will be blocked.
- If you use TLS for coturn, add turn: URL with 5349 and transport=tcp.
